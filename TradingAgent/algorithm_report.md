## 3. 核心问题诊断

### 问题1: 🔴 推荐器选出了PT（退市）股票

**这是最严重的问题。** 实盘推荐器在 04-23 和 04-24 推荐了：
- `000003 PT金田A` (total_score=-1.75 / -3.7)
- `000015 PT中浩A` (total_score=-1.75 / -3.7)

这些是**已退市的PT股票**，根本没有交易数据。推荐器缺乏：
- 退市/停牌股票过滤
- 负分股票拦截（technical_score=-10 却仍然被推荐）
- 股票状态检查（正常交易 vs PT/ST/退市）

**影响**: 2天×2只 = 4条推荐全是废票，直接拉低实盘胜率。排除这2天后，可用数据只有5天/7只，胜率 5/7=71.4%。

### 问题2: 🟡 回测选股池 vs 实盘选股池完全不同

| 维度 | 回测 | 实盘 |
|------|------|------|
| 选股池 | V19 pool (~4082只) + 3140只有足够K线 | 推荐器候选池（可能只有几十只） |
| 评分方式 | 15+个子分数加权求和 | batch_stock_scores → total_score |
| 分数分布 | 连续分布，区分度好 | 集中在 7.0~7.5，区分度差 |
| 行业信息 | 从batch scores获取 | 多数显示"未知" |
| 市值/换手率 | 从K线数据实时计算 | 硬编码为"50.0亿"/"3.0%" |

**回测可以从3000+只股票里选Top3，实盘只能从推荐器给的候选里选。**

### 问题3: 🟡 评分映射断裂

推荐器的评分公式：
```
total_score = (tech * w1 + chip * w2 + sector * w3 + news * w4) / 某种归一化
```

回测的评分公式：
```
score = momentum_s * w_momentum + trend_s * w_trend + ... + 多个bonus/penalty
```

**这两套评分体系完全不对应。** 回测优化出的权重参数，无法直接应用到推荐器。

### 问题4: 🟠 R2 和 R4/5 优化效果差

- R2: Period A=0%, B=0%（0天数据）—— 说明风险等级2在训练期几乎不出现
- R4/5: Period A=0%, D=0%，B=75%, C=38% —— 优化空间有限

**根本原因**: 市场大部分时间是 risk=3，R2/R4/5 的训练样本太少。

### 问题5: 🟡 样本量严重不足

实盘验证只有 **11只股票/7天**，统计上完全不显著。即使是随机的，11次试验中5次成功的95%置信区间是 [16%, 75%]。45.5%可能只是运气差。

## 5. V24 优化方案

### 方案概述

与其继续优化回测内参数（在3000只股票池上的择优能力），不如解决**回测→实盘的映射问题**。

### 核心改进1: Walk-Forward 验证（消除训练泄露）

当前：4个Period联合优化，Period D被用于训练（泄露未来信息）

V24：
```
训练: Period A + B (02/07 ~ 03/28)
验证: Period C (03/28 ~ 04/22) — 选超参数
测试: Period D (04/22 ~ 05/09) — 只评估一次，不用于调参
```

### 核心改进2: 推荐器对齐评分 (Score Alignment)

不改变推荐器代码，而是在回测中加入**推荐器模拟层**：
- 模拟推荐器的候选生成过程
- 使用与推荐器相同的评分公式
- 回测优化直接优化推荐器能用的参数

### 核心改进3: 最低分数阈值 (Confidence Filter)

```python
MIN_SCORE_THRESHOLD = 7.0  # 低于此分数不推荐
# 如果当天没有股票超过阈值，则不交易（保住本金）
```

这是最重要的实盘改进——不交易比乱交易好。

### 核心改进4: 状态过滤器 (Stock Status Filter)

```python
def is_tradeable(code, name, static_data):
    """过滤不可交易股票"""
    # PT/ST/退市
    if 'PT' in name or 'ST' in name: return False
    if code.startswith('2') or code.startswith('9'): return False  # B股
    # technical_score < 0
    if static_data.get('tech', 5) < 0: return False
    # 无行业信息
    if static_data.get('industry', '未知') == '未知': return False
    # 成交量/换手率异常
    return True
```

## 8. 结论

| 问题 | 严重程度 | 修复方式 |
|------|----------|----------|
| PT股票被推荐 | 致命 | 推荐器加过滤 |
| 选股池不对齐 | 重要 | 推荐器模拟层 |
| 评分映射断裂 | 重要 | V24 统一评分 |
| R2/R4/5样本少 | 次要 | V24已合并为统一模型 |
| 样本量不足 | 重要 | 需更多实盘数据 |

**优先级**: 
1. 先修推荐器的PT过滤和最低分数阈值
2. 让推荐器使用V24的评分公式（或至少对齐特征工程）
3. 继续积累实盘数据验证

**V24回测结果诚实可靠**: 69.2%测试集beat_idx，没有数据泄露，没有过拟合。
