# V5 优化分析报告

## 执行摘要

经过深入分析和多轮实验，**V4的66.7% beat_index已接近当前数据和方法的性能天花板**。要达到85%目标，需要从根本上扩充数据源或改变方法论。

| V5-d | 混合(V4-R3 + 超额-R4+) | 19/33 = 57.6% | 63.6% | R4=88%, R3下降 |

## ML模型训练

使用LogisticRegression训练（13k样本, 16个特征）:

### 特征重要性
1. rs5 (+1.95): 相对强度5d是最强正预测因子
2. r5 (-1.87): 原始动量反而最负(超买)
3. rs3 (+1.27): 相对强度3d
4. r3 (-1.16): 原始3d动量

### ML结果
- Walk-forward测试（前15天训练，后18天测试）: **22.2%** (远差于V4)
- 全部数据: 42.4%
- 结论: 线性模型严重过拟合，无法推广

### 失败根因

1. **R5单股集中风险(4/7天失败)**: V4在R5只选1只股票，运气成分太大
2. **指数上涨日选股不力(7/11天)**: 指数上涨但选的股票没跟上
3. **全仓亏损日(5天)**: 所有选股都亏损，选股器完全误判方向
4. **黑名单太温和**: 连续亏损股可能被重新选入

## 关键洞察

1. **V4评分已接近理论极限**: combo_mom_vol因子(最简单)也得66.7%
2. **更多股票≠更好**: N=3是最优,N>3时额外股票拉低收益
3. **相对强度>原始动量**: ML发现rs5是最佳预测因子,r5反而是负的
4. **低波动策略陷阱**: vol5=0的股票是停牌/不活跃股,不是防御
5. **防御策略反效果**: 在R5日,75%的股票涨但防御模式选的股票反而跌

## 达到85%需要的改变

### 数据层面(最关键)
1. **扩充股票池**: 899→3000+只,增加选股空间
2. **更新K线数据**: 当前截止4/7,需更新到最新
3. **增加北向资金/融资融券数据**
4. **行业板块数据补充**: 当前只5只股票有评分数据

### 算法层面
1. **Walk-forward权重优化**: 滚动窗口优化评分权重
2. **非线性模型**: XGBoost/LightGBM需要更多训练数据
3. **因子组合**: 用IC测试找到真正predictive的因子
4. **择时模块**: 识别极端日(如R5)的子类型(继续跌vs反弹)

### 策略层面
1. **空仓机制**: 需要更好的预测(当前简单空仓亏钱)
2. **行业轮动**: 追踪板块轮动信号
3. **持仓管理**: 多日持仓+动态调仓
4. **扩展评估期**: 33天太少,65+天更稳定
